# OSLO. Experiments

> Dependencies: `oslo-design-system` (non-negotiable even in experiments).
> Innovation within discipline — never abandon brand DNA.

---

## Purpose

- Criar interfaces conceituais, testes de UX e visual explorations sem romper com o DNA da OSLO
- Explorar futuro com sofisticação e disciplina
- O experimento deve ampliar a marca, não descaracterizá-la

---

## The Core Rule

> **Toda experimentação deve continuar parecendo premium, estratégica e plausível.**
> Nunca transformar a OSLO em uma marca sci-fi genérica.

---

## What's Allowed

| Category | Examples |
|---|---|
| Data visualizations | Interactive architecture maps · AI ecosystem visualizations |
| Motion / animation | Animated workflow diagrams · narrative page transitions |
| Product concepts | Modular service configurators · agent-based interface ideas |
| Data interfaces | Premium data interfaces · multi-agent dashboards |
| Spatial UX | Process intelligence maps · system orchestration views |
| Prototypes | AI product onboarding flows · minimal premium prototypes |
| Storytelling | Business + technology narrative experiences |

---

## Visual Boundaries

### Always Maintain
- Black dominant canvas
- White Serene text (`#fffafa`)
- Azure as restrained accent (`#275279`)
- Montserrat for headlines
- Inter for UI text
- Clean geometry

### Always Avoid
- Neon gradients
- Purple/cyan sci-fi palettes
- Heavy glassmorphism
- Playful startup illustrations
- Cartoon icon systems
- Overanimated interfaces
- Cyberpunk / hologram aesthetics

---

## Acceptable Moods

| Mood | What It Means |
|---|---|
| Architectural | Structural, grid-based, precise |
| Strategic | Shows system relationships and logic |
| Refined | Elevated aesthetic, premium feel |
| Intelligent | Shows reasoning, patterns, connections |
| Premium-tech | High-end tools for serious work |
| Calm confidence | Control without aggression |

---

## Animation Guidelines for Experiments

**Allowed:**
- Smooth entrance animations (opacity + translate, 300–600ms)
- Data flowing through system diagrams
- Cards that reveal depth on hover
- Subtle particle or network effects (sparse, monochromatic)
- Progress indicators that feel surgical

**Not allowed:**
- Bouncy spring animations
- Spinning 3D objects without purpose
- Glitch effects
- Colorful particle explosions
- Looping backgrounds that distract from content

---

## Interaction Patterns for Experiments

**Interesting and on-brand:**
- Node-and-edge system maps that reveal on hover
- Step-by-step architecture builders
- "Before / After" operational state comparisons
- Real-time metric simulations with Azure highlights
- Expandable process flow diagrams
- AI agent activity visualizations

---

## Language in Experiments

Use the language of:
- Systems · architecture · intelligence · operations · precision
- Describe interactions with intentionality
- Prefer visual sophistication over novelty for novelty's sake
- No startup excitement — calm technical confidence

---

## Experiment Quality Test

Before shipping any experiment, ask:
1. Does this still look like it belongs to OSLO?
2. Would a Fortune 500 CTO find this impressive, not playful?
3. Does the black canvas dominate?
4. Is Azure used sparingly?
5. Is there a clear strategic idea behind the interaction?
6. Does it amplify the brand or distract from it?

If any answer is uncertain — pull back, not forward.
