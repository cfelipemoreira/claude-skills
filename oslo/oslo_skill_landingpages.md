# OSLO. Landing Pages

> Dependencies: read `oslo-design-system`, `oslo-ui-components`, `oslo-language`
> before building any page. These three skills define the visual and verbal rules.

---

## Page Principles

- Cada página deve vender clareza, não exagero
- A landing precisa parecer consultiva, técnica e premium
- Texto direto, visual limpo, arquitetura forte

---

## Canonical Sections

### 1. Hero (Required)
**Required elements:**
- Headline (curta, forte, conceitual)
- Subheadline (explica claramente o que a OSLO faz)
- Primary CTA

**Optional elements:**
- Secondary CTA
- Credibility line
- Abstract system visual

### 2. Who We Are
- Goal: explicar rapidamente quem é a OSLO e qual interseção estratégica ocupa
- Falar de IA + estrutura + operação + negócio
- Evitar explicações longas e professorais

### 3. Solutions
- Cards: AI Integration · AI Building · Training/Advisory (when applicable)
- Each card: problema resolvido + tipo de entrega + impacto

### 4. Methodology
- Structure: Diagnóstico → Automação → Inovação
- Explicar método como processo inteligente, não como burocracia

### 5. Use Cases
Help the lead recognize themselves:
- Equipes com ferramentas desconectadas
- Operações lentas
- Falta de padronização
- Necessidade de aplicações sob demanda
- Desejo de ganho de eficiência e escala

### 6. Outcomes
Translate delivery into perceived result:
- Mais eficiência · menos fricção operacional · mais clareza de processo
- Mais velocidade · melhor integração entre áreas · estrutura para crescer

### 7. Proof
- Selected clients · brand logos · case snapshots · operational metrics
- Pouca prova, mas boa — nunca parecer inventado

### 8. FAQ
Themes: para quem é · como funciona a contratação · o que entregamos ·
o que não fazemos · prazo/complexidade · integrações/sistemas

### 9. Contact CTA
- Goal: converter com elegância
- CTA firme, sem parecer agressivo
- Reforçar conversa estratégica

## Page Types

### Institutional Home
- **Objective:** Apresentar a OSLO de forma ampla
- **Tone:** conceptual + commercial
- **Focus:** positioning · solutions · methodology · credibility

### Integration Page
- **Objective:** Vender integração de sistemas, automações e ecossistemas operacionais
- **Focus:** stack integration · workflow intelligence · operational efficiency · AI-based process layer

### Building Page
- **Objective:** Vender desenvolvimento de aplicações e sistemas sob demanda
- **Focus:** custom systems · AI-native products · internal tools · strategic product development

### Training Page
- **Objective:** Vender mentorias, treinamentos ou capacitação aplicada
- **Focus:** practical AI adoption · team enablement · strategic use of AI

### Case Study Page
- **Structure:**
  1. Context
  2. Challenge
  3. Diagnosis
  4. Solution Architecture
  5. Implementation
  6. Impact

## Copy Structure Rules

| Element | Rules |
|---|---|
| Headlines | curtas · fortes · sofisticadas · sem buzzwords demais |
| Subheadlines | claras · explicativas · orientadas a valor |
| Body | frases curtas a médias · menos adjetivo, mais substância |
| CTAs | "Entrar em contato" · "Falar com a OSLO." · "Entender a solução" · "Ver como funciona" |

---

## Section Sequencing (Recommended)

**Institutional Home:**
Hero → Who We Are → Solutions → Methodology → Use Cases → Outcomes → Proof → FAQ → Contact CTA

**Service Page:**
Hero → Problem Definition → Solution Architecture → How It Works → Outcomes → Cases/Proof → FAQ → Contact CTA

**Case Study:**
Context → Challenge → Diagnosis → Architecture → Implementation → Impact → CTA

---

## Quality Checklist

Before finalizing any page:
- [ ] Hero headline is short, strong, and concept-driven
- [ ] Subheadline clearly explains what OSLO does
- [ ] Each section has one central idea
- [ ] No buzzwords or startup hype language
- [ ] Azure used sparingly as accent only
- [ ] All CTAs are calm and firm, not aggressive
- [ ] Proof feels real and grounded
- [ ] Mobile layout breathes properly
- [ ] Typography hierarchy follows oslo-design-system
