# OSLO. Proposals & Commercial Documents

> Dependencies: `oslo-language` for tone · `oslo-knowledge` for service descriptions.
> Proposals are sales instruments — they must be clear, structured, and confident.

---

## Proposal Philosophy

- Uma proposta da OSLO deve parecer uma entrega em si mesma
- Mostra arquitetura de pensamento antes da arquitetura técnica
- Transmite: nós entendemos seu problema melhor do que você esperava
- **Nunca parece um template genérico preenchido**

---

## Proposal Types

| Type | When to Use |
|---|---|
| Discovery Proposal | Primeira conversa comercial — mapear problema e propor diagnóstico |
| Integration Proposal | Projetos de automação, integrações e AI Integration |
| Building Proposal | Desenvolvimento de aplicações, produtos ou sistemas |
| Retainer Proposal | Projetos contínuos com fee fixo + variável |
| Strategic Advisory | Mentorias, capacitação, strategy sessions |

## Standard Proposal Structure

### 1. Context
> O que entendemos sobre a situação atual do cliente.
- Resumo do contexto operacional
- Cenário de mercado relevante
- O que o cliente está tentando resolver

### 2. Diagnosis
> Nossa leitura do problema real.
- Raiz do problema
- Impacto atual (operacional, financeiro, estratégico)
- O que acontece se nada mudar

### 3. Proposed Architecture
> Como a OSLO vai resolver.
- Visão geral da solução
- Componentes principais (sistemas, integrações, aplicações)
- Como os elementos se conectam

### 4. Scope of Work
> O que será entregue.
- Fase 1, Fase 2, Fase 3 (se aplicável)
- Deliverables por fase
- O que está fora do escopo

### 5. Method
> Como trabalhamos.
- Referência ao OSLO Project Method (Diagnosis → Architecture → Development → Implementation → Optimization)
- Cadência de trabalho
- Comunicação e alinhamento

### 6. Investment
> Estrutura de investimento.
- Modelo: On Demand ou Fixed Fee + Variable
- Valores (se aprovado para inclusão)
- Condições de pagamento

### 7. Next Steps
> Como avançar.
- Ação clara para o cliente
- Prazo de validade da proposta (se aplicável)
- Contato para dúvidas

## Writing Rules for Proposals

**Tone:** consultive · confident · precise · premium
**Length:** concise — each section should be scannable in 30 seconds
**Language:** follow `oslo-language` rules completely

| Element | Rule |
|---|---|
| Section headers | Clear and substantive, not generic |
| Problem description | Specific, shows understanding |
| Solution description | Architectural, not feature-list |
| Scope items | Concrete deliverables, not vague outcomes |
| Investment | Clear structure, no ambiguity |
| CTAs | Firm and calm — "O próximo passo é..." |

---

## Scope of Work Format

```
Phase 1 — [Phase Name]
Duration: X weeks
Deliverables:
  - [Specific deliverable]
  - [Specific deliverable]

Phase 2 — [Phase Name]
Duration: X weeks
Deliverables:
  - [Specific deliverable]
```

---

## Investment Section Options

### On Demand
```
Investment: R$ [value]
Model: Fixed fee per project
Includes: [list of inclusions]
Does not include: [list of exclusions]
```

### Fixed Fee + Variable
```
Monthly fee: R$ [value]
Variable component: [description of variable]
Minimum commitment: [X months]
```

## Language Patterns for Proposals

**Opening context:**
> "A [Company] opera em um cenário onde [context]. O desafio central é [problem], o que impacta diretamente [consequence]."

**Proposed solution:**
> "A OSLO propõe a arquitetura de um sistema que [solution description], integrando [components] para entregar [outcome]."

**Closing:**
> "Se faz sentido avançar com esse diagnóstico, o próximo passo é [action]. Estamos disponíveis para alinhar os detalhes."

---

## What to Avoid in Proposals

- Listas genéricas de tecnologias sem contexto
- Promessas vagas de "transformação digital"
- Preços sem estrutura clara
- Escopos abertos demais
- Tom professoral ou excessivamente técnico
- Templates que parecem templates
